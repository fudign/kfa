# Prime Prompt: Technical Design Document

Create technical design document for new features or systems.

## Usage

```bash
kfa prime use documentation/technical-design-doc "Design real-time notification system"
```

## Prompt Template

I need a technical design document for:

**Context:** {CONTEXT}

### Document Structure

1. **Overview**
   - Problem statement
   - Goals and non-goals
   - Success metrics
   - Timeline

2. **Background**
   - Current state
   - Limitations
   - Why change is needed

3. **Proposed Solution**
   - High-level approach
   - Architecture diagram
   - Key components
   - Data flow
   - API design

4. **Alternatives Considered**
   - Other approaches
   - Pros/cons
   - Why not chosen

5. **Technical Details**
   - Database schema
   - API endpoints
   - Algorithms
   - Third-party services
   - Security considerations
   - Performance considerations

6. **Implementation Plan**
   - Phases
   - Dependencies
   - Rollout strategy
   - Testing strategy

7. **Risks and Mitigation**
   - Technical risks
   - Mitigation strategies
   - Contingency plans

8. **Open Questions**
   - Unresolved issues
   - Need for discussion

## Success Criteria
- ✅ Problem clearly defined
- ✅ Solution well-explained
- ✅ Alternatives considered
- ✅ Implementation plan clear
